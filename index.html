Atelier 1 : L'Authentification Basic

L'Authentification Basic permet de vérifier vos nom d'utilisateur et mot de passe chaque fois que vous demandez à avoir accès un répertoire Web protégé.

Accéder à l'atelier 1 : L'authentification basic (Les exercices de cet atelier vous seront fournis par votre enseignant)


Atelier 2 : L'authentification par Cookies

Les cookies d'authentification sont couramment utilisés par les serveurs Web pour authentifier qu'un utilisateur est connecté et avec quel compte il est connecté.

Accéder à l'atelier 2 : L'authentification par Cookies (Les exercices de cet atelier vous seront fournis par votre enseignant)


Atelier 3 : L'authentification via les Sessions

L'authentification permet de s'assurer que seuls des utilisateurs légitimes peuvent accéder à l'application tandis que le mécanisme des sessions assure le suivi des diverses actions réalisées par les utilisateurs sur l'application.

Accéder à l'atelier 3 : L'authentification via les Sessions (Les exercices de cet atelier vous seront fournis par votre enseignant)


Atelier 4 : L'authentification via les requêtes HTTP

L'authentification de base est codée dans la requête HTTP qui contient le message SOAP. Lorsque le serveur d'applications reçoit la requête HTTP, le nom d'utilisateur et le mot de passe sont extraits et vérifiés à l'aide du mécanisme d'authentification spécifique au serveur.

Accéder à l'atelier 4 : L'authentification via les requêtes HTTP (Les exercices de cet atelier vous seront fournis par votre enseignant)
